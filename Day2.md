### Day 2- Software Development Life Cycle
Software development life cycle is well organized phase of building any kind of system. It is the structured framework that guides the process  of planning, creating, testing, deploying, and maintaining software. Its  primary goal is to deliver high-quality, cost effective, and maintainable software that meets user requirements within defined timelines and budgets.

## Phases of Software Development Life Cycle

# 1. Requirement Gathering
This is the foundational phase where the team figures out exactly what needs to be built.

What happens: Developers, project managers, and clients talk to each other to collect all the requirements. For example, if you are building a Student Management System, you figure out what information needs to be stored (like student names, IDs, grades) and what features are needed (like adding, updating, or deleting a student record).

Outcome: A clear document (often called an SRS or Software Requirement Specification) that lists everything the software must do.

---

# 2. Planning
Once the requirements are clear, the team plans how to execute the project.

What happens: The team calculates the cost, decides on the timeline, determines how many people are needed, and identifies potential risks. They also choose the technologies (like coding languages and tools) they will use.

Outcome: A solid project plan and timeline.

---

 # 3. System Design
Before writing any code, you need a blueprint of how the software will look and work.

What happens: Senior developers and architects design the structure of the software. They plan how data will flow, how the database tables will look, and how the user interface (UI) will appear.

Outcome: Design documents, flowcharts, and architectural diagrams.
---
# 4. Coding / Implementation (Labeled as "Deployment" at the bottom right)
This is where the actual building happens.

What happens: Developers take the design blueprints and start writing the actual code using a programming language (like C, C++, or Java). This is typically the longest phase of the cycle.

Outcome: A working software prototype or source code.
---
# 5. Testing
Once the code is written, it must be thoroughly checked to make sure it works correctly without breaking.

What happens: Quality Assurance (QA) engineers and testers run the software to find bugs, glitches, or errors. They check if the software actually behaves the way it was supposed to based on the "Requirement Gathering" phase. If they find a bug, they send it back to the developers to fix.

Outcome: A stable, bug-free software product.
---
# 6. Deployment
Now that the software is tested and working perfectly, it is ready for the real world.

What happens: The software is released and installed for the end-users to use. For a mobile app, this means putting it on the App Store; for an enterprise system, it means installing it on the client's servers.

Outcome: The software is live and active.
---
# 7. Maintenance
The process doesn't end just because the software is released.

What happens: Once real people start using the software, unexpected issues might pop up, or users might request minor updates. During maintenance, developers fix newly discovered bugs, improve performance, and update the software to ensure it keeps running smoothly on newer devices or operating systems.
