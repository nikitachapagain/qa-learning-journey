# 🚀 Day 7 – Learning Bug Reporting in Manual Testing

Today I learned one of the most important skills in Manual Testing: **how to write a proper Bug/Defect Report**. A well-written bug report helps developers understand the issue quickly and fix it efficiently.

## 📖 Topics Covered

- What is a Bug/Defect Report?
- Bug Report Format
- Severity vs Priority
- Environment Types
- Expected Result vs Actual Result
- Bug Status
- Bug Attachments

## 📝 Bug Report Format

A standard bug report includes:

1. Bug ID
2. Title/Summary
3. Project/Module
4. Environment
5. Severity
6. Priority
7. Description
8. Steps to Reproduce
9. Expected Result
10. Actual Result
11. Status
12. Assigned To
13. Attachments (Screenshots/Logs)
14. Reported By
15. Date

## 🌍 Environment Types

- **Local** – Used by developers during development.
- **QA** – Used by testers for testing.
- **UAT (User Acceptance Testing)** – Used by end users before release.
- **Production (Prod)** – Live environment used by customers.

## ⚡ Severity vs Priority

### Severity
Severity defines **how much impact** a bug has on the system.

- 🔴 Critical – Application crashes or becomes unusable.
- 🟠 Major – Main functionality is not working.
- 🟡 Minor – Small functionality issue with a workaround.
- 🟢 Trivial – UI or cosmetic issues.

### Priority
Priority defines **how urgently** the bug should be fixed.

- 🔴 High – Fix immediately.
- 🟡 Medium – Fix in the next release/sprint.
- 🟢 Low – Can be fixed later.

> 💡 A bug can have **High Priority but Low Severity**, or **High Severity but Low Priority**, depending on business needs.

## 📌 Expected Result vs Actual Result

- **Expected Result:** What the application should do.
- **Actual Result:** What actually happens when the test is executed.

## 🔄 Common Bug Status

- To Do
- In Progress
- In Review
- Fixed
- Retest
- Closed
- Reopened

## 💡 Key Takeaways

- A clear bug report saves developers' time.
- Severity measures technical impact, while Priority measures business urgency.
- Always include clear reproduction steps and screenshots whenever possible.
- Good communication is just as important as finding bugs.
